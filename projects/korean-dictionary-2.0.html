2. Inspiration and Concept

Naver Dictionary used to have a feature where you would search for a word and they would search for instances of that word within show transcripts or closed captions in their database. While this was in theory a good idea, the examples never had sufficient context to make sense.

In my original Korean dictionary, users can add example sentences for any word. This is useful, but it requires both a lot of time and conscious awareness that you're seeing an unfamiliar word. Words you do not take notice of fly under the radar.

If you could instead let the computer add the examples for you, it would add (in theory) ALL of the examples, even for these under-the-radar words. If you ever do take notice of them, you'll have already collected a set of contextual examples for the word. And since you were the one who added those examples, they will make a whole lot more sense.

In addition to this feature, expanding user options and user experience in other ways has been the goal of this second dictionary. It has improved UI and improved settings and customizability. I also rewrote the entire project in Typescript to make the project more maintainable.

I am currently working on the auto-derived examples system for the project. This has involved creating a tool that picks the correct headword out of a list (see projects/headword-disambiguator).
