Headword Finder

  • Words underlined like this are defined when hovered over.

1. Summary

I made this as a requirement for my second Korean dictionary. The goal is to be able to determine which of a lemma's headwords is being used in a specific context. For example, if given the sentence 'The pen has no ink left', I need to be able to determine whether the word 'pen' in this sentence refers to a writing utensil pen or a fenced area in which livestock are kept pen. After testing various methods of picking a headword, I found a model that has accuracy suitable for the purposes of the new Korean dictionary.

2. Inspiration

A major goal of the second Korean dictionary is to implement the ability to automatically derive example sentences from any text.

For greater precision when showing these examples, the implementation of this feature should also attempt to figure out the meaning of the words in an example sentence. This way, it can only show the example for the correct headwords rather than all of the headwords corresponding to a lemma.

To reuse the example from the Summary section, assume a user inputs the text 'The pen has no ink left'. The system should automatically detect the lemma 'pen' in this sentence and place the sentence in the correct dictionary entry. It should appear under the headword refering to the writing utensil, but not under the headword refering to an animal enclosure.

For many lemmas, only one headword is frequently used. In these cases, it is prudent to simply pick the common headword. However, cases like pen above are numerous. For this project, not making any decision (saying 'not sure') when no headword seems especially likely to be correct is fine.

However, there needs to be a balance between making decisions often enough to fulfill its role as a useful component of the dictionary and between being accurate in its guesses.

This project involved finding a way to use a model like KoBERT to find this balance. Using KoBERT and various parameters, I can return a response around 45% of the time with around 94% accuracy for ambiguous usages of the tested lemmas. Longer texts generally net better results.

For unambiguous lemmas, the model will readily default to the common headword as intended. These easy-to-get-correct cases also make the overall response rate greater than 45% and the accuracy greater than 94%.
